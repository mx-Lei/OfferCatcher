# Offer 捕手 —— 学生求职匹配智能体 方案说明

## 一、问题诊断

当前高校学生在求职过程中面临两个核心痛点：

1. **信息不对称**：学生难以判断自身条件与目标岗位的匹配程度，往往盲目投递，导致通过率低下。
2. **简历优化缺乏针对性**：市面上的简历优化工具多为通用模板，无法结合具体岗位 JD（职位描述）给出精准修改建议。

传统解决方案依赖人工咨询，成本高、周期长；现有自动化工具多采用关键词匹配，无法理解语义层面的匹配关系。因此，需要一套基于大语言模型的智能匹配与优化系统，帮助学生低成本、高效率地完成求职准备。

---

## 二、方案设计

本系统命名为「Offer 捕手」，是一个基于 Streamlit 构建的 Web 智能体，核心流程如下：

1. **简历上传与解析**：用户上传 PDF 或 Word 格式简历，系统自动提取纯文本。
2. **目标岗位设定**：用户可输入任意岗位名称，系统从内置岗位库中匹配；若库中没有，则自动生成虚拟岗位（含通用 JD）用于后续分析。
3. **AI 语义匹配**：将简历文本与岗位列表一并提交给大模型，返回每个岗位的匹配分数及优劣势分析。
4. **优化建议生成**：针对匹配结果中用户关注的岗位，大模型生成 3-5 条具体修改建议，每条包含问题描述、修改建议和可套用示例。

系统采用纯 Python 技术栈，基于 Streamlit 构建 Web 界面，开发效率高，部署流程简洁。

---

## 三、AI 工具选型理由

| 工具 | 选型理由 |
|------|----------|
| **Streamlit** | 纯 Python 写 Web 界面，无需 HTML/CSS/JS；`pip install streamlit` 一行启动，部署简单（支持 Streamlit Cloud 托管）。 |
| **硅基流动 DeepSeek-V3 API** | 兼容 OpenAI SDK，调用方式熟悉；中文理解能力强，适合处理简历与岗位 JD 的语义匹配；按量计费，成本低。 |
| **PyMuPDF（fitz）** | PDF 解析速度快，保留段落结构，对中文简历兼容性好。 |
| **python-docx** | Word 简历的标准解析库，可提取段落、表格等多种元素。 |

---

## 四、关键配置

- **API 密钥管理**：优先读取 Streamlit Secrets（`st.secrets["API_KEY"]`），本地运行时支持侧边栏手动输入，兼顾部署安全性与本地调试便利性。
- **岗位数据**：`jobs.json` 内置 5 个真实岗位（前端/后端/数据分析/产品/算法），同时支持用户输入任意自定义岗位，由 LLM 动态生成虚拟 JD，彻底取消对固定岗位库的强制依赖。
- **会话状态**：使用 `st.session_state` 管理解析文本、匹配结果、优化建议，避免页面刷新导致数据丢失。
- **部署**：通过 GitHub 仓库 `mx-Lei/OfferCatcher` 自动部署至 Streamlit Cloud，推送即更新。

## 六、效果评估

**功能完整性**：系统支持简历解析、岗位匹配、优化建议三大核心功能，覆盖学生求职准备的主要场景。

**易用性**：用户仅需上传简历、输入目标岗位两步操作，无需注册登录，界面简洁，学习成本低。

**扩展性**：自定义岗位功能使得系统不依赖固定岗位库，可适用于任意行业、任意职位的匹配分析。

**部署可行性**：已成功部署至 Streamlit Cloud，获得公网访问链接，可在线演示完整功能。

**待改进方向**：
- 匹配结果目前为单次调用，可引入多轮对话让用户可以追问细节；
- 岗位库可扩展为在线爬取或用户自定义上传；
- 可加入面试问题生成功能，进一步提升求职准备完整度。
